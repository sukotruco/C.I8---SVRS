04 · Background

System Context

The sanding systems are strategically integrated into the brake architecture to optimise adhesion management across the Northern fleet.

Sanding equipment is located on the DMSL and DMS cars, mounted on the secondary bogie of each car. For each designated bogie, the discharge mechanism is installed on a single-axle basis, ensuring precise delivery to the wheel-rail interface of the leading axle.

The current Northern fleet system uses a fixed, non-speed-dependent sand flow rate (per GMRT 2045), calibrated during dynamic commissioning. The new SVRS system replaces this with intelligent, variable-rate control managed by Knorr-Bremse.

Sanding activation logic

  • Automatic in emergency braking until train is stopped
  • Automatic in braking mode whenever WSP is active
  • Manual demand under driver request — traction only (5-second burst)
  • Inhibited if sand level drops below 25% in traction mode
  • TCMS activates sanding valve when any BCU detects WSP activity or emergency loop opens above 10 km/h

05–06 · Proposals

Integration Architecture

Knorr-Bremse proposes centralised intelligent sanding control. CAF integrates via hardwired signals and MVB network from cabinet C2 of each cab car.

Knorr-Bremse
SVRS Control Rack
Centralised control of sanding — automatic, manual, and test modes managed within the SVRS equipment. TCMS functions exclusively as a data provider.
  • Control rack installed in cabinet C2 (DMSL & DMS)
  • Autonomous processing of velocity, WSP and isolation signals
  • SVRS fully responsible for all pneumatic valves in sanding panel
  • Cross-cab synchronisation via MVB monitoring
CAF Integration
Vehicle Integration
CAF interfaces the KNBS system via hardwired +110 Vdc signals and the vehicle MVB network. Plug-and-play via pin coding on I/O controller connectors.
  • 12 hardwired signals supplied (direction, WSP, brake, cab active…)
  • Full MVB signal list per Appendix 1 (speed, bogie data, flow rate)
  • TCMS logic modification: new CCU logic + HMI alarm management
  • LeadMind® database update for shore-side data parity
07 · Hardware

System Components

Seven hardware elements constitute the complete SVRS installation. KNBS supplies; CAF installs and integrates.

7.1
Pneumatic Panel
Retrofit of pneumatic components in situ by KNBS without dismantling the panel. Modification required to implement variable sanding functionality.
KNBS Retrofit · In-situ
7.2
SVRS IO Module
Two racks installed in C2 cabinets of DMSL and DMS. Each controls the SVRS system for a pair of sanders per direction of travel.
Cabinet C2 · 2× per vehicle
7.2.1
Sand Level Sensor
Installed at top of each hopper. Supplied by KNBS, installed by CAF. Dynamically measures sand level, mitigating erroneous readings from vehicle motion.
KNBS Supply · CAF Install
7.2.2
Sand Hopper
Existing hoppers retained. Dismantled by CAF for machining to accommodate level sensor and SDG valve. No structural replacement required.
Existing · CAF Machining
7.2.3
SDG Valve
New sanding valve supplied by KNBS, dynamically controlled by the KNBS rack. Same mechanical interface as current SDN31.1 — no additional hopper modification.
SDN31.1 Compatible
7.2.4
Sand Flow Sensor
Integrated to monitor sand discharge performance. IEC 61373 Cat3 compliant for shock and vibration. Data transmitted to TCMS via discrete wiring or MVB.
IEC 61373 Cat3 · MVB
7.2.5–6
Heated Nozzle & Drying Heater
Remain under TCMS control. Wiring and connectors adjusted for the new SDG valve assembly interface. No change to baseline vehicle logic.
TCMS Managed · Unchanged logic
7.2.7
Air Pipes
Discharge pipe shortened by ~300mm at sanding unit interface. Bespoke adapter pipe or reinforced hose bridges legacy piping to new SDG sanding unit.
−300mm · Adapter pipe

07.4 · Software

TCMS & Software Updates

Three on-board software components require update. TCMS continuously monitors SVRS communication status; failure triggers HMI alarm.

New TCMS Logic (CCU)
Updated Central Control Unit logic to integrate comprehensive SVRS functional control, including communication monitoring with both units and fault handling.
Alarm Management (HMI)
Updated alarm management version covering 6 new alarms (AL6305–AL6310) and 5 modified existing alarms. Reports to driver, operating crew and maintenance personnel.
Marshalling Config (HMI)
Updated marshalling configuration to reflect new SVRS signal map. Additionally, LeadMind® onboard database updated — new SDIAG software version required.
